## GATE UI-1 — ACCESSIBILITY (WCAG 2.2) — CRITICAL

| Check | Rule | Anti-Pattern |
|-------|------|--------------|
| **color-contrast** | Min 4.5:1 normal text, 3:1 large text (WCAG AA); 7:1 for AAA | `#999` on white (2.8:1) |
| **focus-states** | Visible focus rings 2-4px on ALL interactive elements | `outline-none` without alternative |
| **touch-target-size** | Min 44x44pt (iOS) / 48x48dp (Android); extend hit area beyond visual bounds | `w-6 h-6` buttons |
| **touch-spacing** | Min 8px gap between adjacent touch targets | Touch targets touching |
| **keyboard-nav** | Tab order matches visual order; full keyboard operability; no traps | Unreachable elements |
| **form-labels** | Use `<label for>` attribute; NEVER placeholder-only | `placeholder='Email'` as only label |
| **aria-labels** | `aria-label` for icon-only buttons; meaningful SR labels | `<button><Icon/></button>` |
| **reduced-motion** | Respect `@media (prefers-reduced-motion: reduce)` | Forced animations |
| **color-not-only** | NEVER convey info by color alone — add icon/text | Red border only for error |
| **heading-hierarchy** | Sequential h1→h6, no level skip | h1 then h4 |
| **alt-text** | Descriptive alt text for meaningful images; empty alt for decorative | `alt=''` for content images |
| **skip-links** | "Skip to main content" link for keyboard users | No bypass for navigation |

### Quick Detection
```bash
# Search for common accessibility violations
grep -rn 'outline-none' src/          # Check for focus ring removal
grep -rn 'placeholder=.*only' src/    # Placeholder-only inputs
grep -rn 'aria-hidden.*true' src/     # Hidden from SR (verify intentional)
grep -rn 'onClick.*<div' src/         # Div as button (needs role/button)
grep -rn 'role=.*img.*aria-label' src/ # Good: img with label
```

### Fix Patterns
```tsx
// ❌ BAD: Icon-only button without label
<button><CloseIcon /></button>

// ✅ GOOD: With aria-label
<button aria-label="Close dialog"><CloseIcon /></button>

// ❌ BAD: No focus ring
<button className="outline-none bg-blue-500">Click</button>

// ✅ GOOD: Custom focus ring
<button className="focus:ring-2 focus:ring-blue-500 focus:ring-offset-2 bg-blue-500">
  Click
</button>

// ❌ BAD: Touch target too small
<button className="w-6 h-6"><Icon /></button>

// ✅ GOOD: Minimum 44x44
<button className="min-w-[44px] min-h-[44px] flex items-center justify-center">
  <Icon />
</button>
```

## GATE UI-2 — LAYOUT & RESPONSIVE — HIGH

| Check | Rule | Anti-Pattern |
|-------|------|--------------|
| **viewport-meta** | `width=device-width initial-scale=1` (NEVER disable zoom) | Missing/incorrect viewport |
| **mobile-first** | Design mobile-first, enhance upward | Desktop-first → mobile broken |
| **breakpoints** | 375px / 768px / 1024px / 1440px consistent | Single device development |
| **readable-font-size** | Min 16px body text on mobile (prevents iOS auto-zoom) | `text-xs` for body |
| **line-length** | Mobile 35-60 chars; desktop 65-75 chars (`max-w-prose`) | Full-viewport-width paragraphs |
| **no-horizontal-scroll** | No horizontal scroll on mobile | `overflow-x` on body |
| **spacing-scale** | 4pt/8dp incremental system (4/8/16/24/32/48/64) | Arbitrary spacing values |
| **z-index-management** | Defined scale: 10/20/40/100/1000; NEVER `z-[9999]` | `z-[9999]` arbitrary |
| **viewport-units** | `min-h-dvh` not `h-screen` (mobile browser chrome safe) | `h-screen` breaking on mobile |
| **container-width** | Use `max-w-*` for content; don't stretch to full width | Unbounded content width |

### Fix Patterns
```tsx
// ❌ BAD: h-screen breaks on mobile browsers (address bar overlap)
<div className="h-screen">

// ✅ GOOD: Dynamic viewport height
<div className="min-h-dvh">

// ❌ BAD: Arbitrary z-index
<div className="z-[9999]">

// ✅ GOOD: Defined scale
<div className="z-50">  {/* modal */}
<div className="z-20">  {/* dropdown */}
<div className="z-10">  {/* card */}

// ❌ BAD: No line length control
<p>Long paragraph stretching across full viewport width...</p>

// ✅ GOOD: Prose width
<p className="max-w-prose">Long paragraph...</p>
```

## GATE UI-3 — VISUAL HIERARCHY — HIGH

| Check | Rule | Anti-Pattern |
|-------|------|--------------|
| **size-distinction** | At least 1.5x difference for clear hierarchy | Similar sizes for different importance |
| **one-primary-action** | One primary CTA per screen | Multiple competing CTAs |
| **squint-test** | Blur your eyes — hierarchy should still be clear | Everything looks same importance |
| **spacing-importance** | More whitespace = more perceived importance | Crowded layout |
| **position-patterns** | Top-left seen first (LTR); F-pattern/Z-pattern scanning | Important content at bottom |
| **density-isolation** | Isolated elements stand out; grouped = scanned as unit | Everything crammed together |
| **weight-hierarchy** | Bold headings (600-700), Regular body (400), Medium labels (500) | All same weight |

### Fix Patterns
```tsx
// ❌ BAD: All same size/weight
<h1 className="text-lg font-normal">Title</h1>
<p className="text-lg font-normal">Body text</p>
<button className="text-lg font-normal">CTA</button>

// ✅ GOOD: Clear hierarchy
<h1 className="text-3xl font-bold">Title</h1>        {/* Primary */}
<p className="text-base font-normal text-gray-600">Body</p>  {/* Tertiary */}
<button className="text-base font-semibold text-white bg-blue-500">CTA</button>  {/* Secondary */}
```

## GATE UI-4 — ANIMATION & TRANSITIONS — MEDIUM

| Check | Rule | Anti-Pattern |
|-------|------|--------------|
| **duration-tiers** | Micro: 50-100ms, Short: 150-250ms, Medium: 250-400ms, Long: 400-700ms | Instant or >700ms |
| **transform-performance** | Use `transform`/`opacity` ONLY; NEVER animate `width`/`height`/`top`/`left` | Animating layout properties |
| **excessive-motion** | Animate 1-2 key elements per view max | Animate everything |
| **easing** | Ease-out entering, ease-in exiting; AVOID linear for UI | Linear for UI transitions |
| **reduced-motion** | Respect `prefers-reduced-motion` | Ignore motion preferences |
| **stagger-sequence** | Stagger list items 30-50ms each; total under 700ms | All-at-once reveals |
| **exit-faster** | Exit animations ~60-70% of enter duration | Same duration both ways |
| **interruptible** | Animations must be cancellable mid-flight | Block input during animation |

### Fix Patterns
```tsx
// ❌ BAD: Animating layout (triggers expensive repaint/reflow)
<div className="animate-[moveDown_300ms]" style={{ top: '100px' }}>

// ✅ GOOD: Using transform (GPU-accelerated)
<div className="transition-transform duration-200 ease-out translate-y-[100px]">

// ❌ BAD: Linear easing for UI
<button className="transition-all duration-200 linear">

// ✅ GOOD: Ease-out for natural feel
<button className="transition-all duration-200 ease-out">

// ❌ BAD: Ignoring reduced motion
<div className="animate-bounce">

// ✅ GOOD: Respecting reduced motion
<div className="motion-safe:animate-bounce">
```

## GATE UI-5 — LOADING & ERROR STATES — HIGH

| Check | Rule | Anti-Pattern |
|-------|------|--------------|
| **show-immediately** | Show something instantly (NEVER blank screen) | Blank screen >100ms |
| **skeleton-screens** | Match skeleton shape to actual content; shimmer animation | Generic spinner for everything |
| **duration-indicators** | <100ms: no indicator; 100ms-1s: subtle; 1-10s: clear; >10s: detailed progress | Long blocking spinners |
| **error-message-format** | What happened + Why + What to do (NEVER blame user) | "Invalid input" without guidance |
| **error-placement** | Inline per-field + summary at top | Errors only at form top |
| **preserve-input** | NEVER clear form on error | Form resets on validation fail |
| **retry-support** | Offer retry for transient failures | Dead-end error screen |
| **focus-invalid** | Auto-focus first invalid field after submit error | No focus management |

### Fix Patterns
```tsx
// ❌ BAD: Blank screen while loading
{loading ? null : <Content />}

// ✅ GOOD: Skeleton matching content shape
{loading ? (
  <div className="animate-pulse space-y-4">
    <div className="h-6 bg-gray-200 rounded w-1/3" />  {/* Title placeholder */}
    <div className="h-4 bg-gray-200 rounded w-full" />  {/* Body placeholder */}
    <div className="h-4 bg-gray-200 rounded w-2/3" />
  </div>
) : (
  <Content />
)}

// ❌ BAD: Unhelpful error message
{error && <p className="text-red-500">Invalid input</p>}

// ✅ GOOD: Clear error with recovery
{error && (
  <p className="text-red-500" role="alert">
    <AlertIcon className="inline mr-2" />
    Email address "john@" is incomplete. Please enter a full email (e.g., john@example.com).
  </p>
)}
```

## GATE UI-6 — FORMS & FEEDBACK — MEDIUM

| Check | Rule | Anti-Pattern |
|-------|------|--------------|
| **visible-labels** | Label EVERY input; NEVER placeholder-only | Placeholder as only label |
| **inline-validation** | Validate on BLUR (not keystroke); show error after user finishes | Real-time validation on every key |
| **submit-feedback** | Loading → success/error state on submit | No response after submit |
| **loading-buttons** | Disable button + show spinner during async operations | Clickable while loading |
| **input-types** | Use semantic types (`email`, `tel`, `number`) for correct mobile keyboard | All `type="text"` |
| **password-toggle** | Show/hide toggle for password fields | Password always hidden |
| **autofill-support** | Use `autocomplete` attributes | No autocomplete hints |
| **toast-dismiss** | Auto-dismiss toasts in 3-5 seconds; errors persist until resolved | Toasts that never disappear |
| **confirmation** | Confirm before destructive actions; prefer undo over "Are you sure?" | Direct delete without warning |

### Fix Patterns
```tsx
// ❌ BAD: Placeholder as label
<input placeholder="Enter your email" />

// ✅ GOOD: Visible label + placeholder as hint
<label htmlFor="email" className="block text-sm font-medium text-gray-700">
  Email address
</label>
<input
  id="email"
  type="email"
  placeholder="john@example.com"
  autoComplete="email"
  className="mt-1 block w-full"
/>

// ❌ BAD: Button clickable while loading
<button onClick={handleSubmit}>Submit</button>

// ✅ GOOD: Disabled + spinner
<button
  onClick={handleSubmit}
  disabled={isSubmitting}
  className="disabled:opacity-50 disabled:cursor-not-allowed"
>
  {isSubmitting ? (
    <>
      <SpinnerIcon className="animate-spin mr-2" />
      Submitting...
    </>
  ) : (
    'Submit'
  )}
</button>
```

## GATE UI-7 — COLOR & TYPOGRAPHY — MEDIUM

| Check | Rule | Anti-Pattern |
|-------|------|--------------|
| **line-height** | 1.5-1.75 body; 1.2 headings | Tight body text |
| **font-scale** | Consistent modular scale: 12/14/16/18/24/32px | Arbitrary font sizes |
| **color-semantic** | Define semantic tokens (primary, error, surface) NOT raw hex | `bg-[#3b82f6]` arbitrary |
| **dark-mode-colors** | Desaturate 10-20%; off-white text (#E0E0E0) NOT pure white | `#FFFFFF` text on dark (OLED smear) |
| **number-tabular** | Tabular figures for data columns, prices, timers | Proportional numbers in tables |
| **contrast-both-modes** | Test contrast in light AND dark mode independently | Only test one mode |

### Fix Patterns
```tsx
// ❌ BAD: Arbitrary colors
<div className="bg-[#3b82f6] text-[#1a1a1a]">

// ✅ GOOD: Semantic tokens
<div className="bg-primary text-text-primary">

// ❌ BAD: Pure white on dark (causes OLED smearing)
<div className="bg-gray-900 text-white">

// ✅ GOOD: Off-white on dark
<div className="bg-gray-900 text-gray-100">  {/* #F3F4F6 not #FFFFFF */}

// ❌ BAD: Arbitrary spacing
<div className="mt-[13px] mb-[7px]">

// ✅ GOOD: Scale spacing
<div className="mt-3 mb-2">  {/* 12px / 8px */}
```

## GATE UI-8 — NAVIGATION PATTERNS — HIGH

| Check | Rule | Anti-Pattern |
|-------|------|--------------|
| **bottom-nav-limit** | Max 5 items; labels WITH icons | Icon-only nav; >5 items |
| **back-behavior** | Predictable back; preserve scroll/state | Broken back button |
| **deep-linking** | All key screens reachable via URL | Static URLs for dynamic content |
| **nav-label-icon** | BOTH icon AND text label | Icon-only navigation |
| **nav-state-active** | Current location visually highlighted | All links same style |
| **modal-escape** | Clear close/dismiss; swipe-down on mobile | No escape from modal |
| **state-preservation** | Restore scroll, filter, input on back | Reset state on navigation |
| **avoid-mixed-patterns** | Don't mix Tab + Sidebar + Bottom Nav at same level | Confusing nav hierarchy |

## GATE UI-9 — PERFORMANCE — HIGH

| Check | Rule | Anti-Pattern |
|-------|------|--------------|
| **image-optimization** | WebP/AVIF, `srcset`/`sizes`, lazy load non-critical | Unoptimized full-size images |
| **image-dimension** | Declare `width`/`height` or `aspect-ratio` to prevent CLS < 0.1 | No dimensions → layout shift |
| **font-loading** | `font-display: swap`; preload only critical fonts | FOIT (Flash of Invisible Text) |
| **virtualize-lists** | Virtualize lists with 50+ items | Rendering thousands of DOM nodes |
| **main-thread-budget** | Per-frame work under ~16ms for 60fps | Blocking main thread |
| **debounce-throttle** | Debounce scroll, resize, input events | Firing handlers on every event |
| **content-jumping** | Reserve space for async content with `aspect-ratio` or fixed height | Layout shift on load |

## GATE UI-10 — TAILWIND V4 SYNTAX — CRITICAL

| Check | Wrong (v3) | Correct (v4) |
|-------|-----------|--------------|
| **Import** | `@tailwind base/components/utilities` | `@import "tailwindcss"` |
| **Config** | `tailwind.config.js` | `@theme { --color-brand: #3b82f6; }` |
| **PostCSS** | `tailwindcss` | `@tailwindcss/postcss` |
| **Opacity** | `bg-opacity-50` | `bg-red-500/50` |
| **Gradients** | `bg-gradient-to-r` | `bg-linear-to-r` |
| **Shadows** | `shadow-sm` (smallest in v3) | `shadow-xs` (smallest in v4) |
| **Important** | `!bg-red-500` (at start) | `bg-red-500!` (at end) |
| **CSS vars** | `bg-[--brand-color]` | `bg-(--brand-color)` |
| **Transform reset** | `transform-none` | `scale-none rotate-none translate-none` |
| **Custom utilities** | `@layer utilities` | `@utility content-auto` |
| **Container queries** | `md:flex-row` (viewport) | `@container` + `@md:flex-row` (parent) |

### Quick Detection
```bash
# v3 syntax that will BREAK in v4
grep -rn '@tailwind ' src/           # Should be @import
grep -rn 'bg-opacity-' src/          # Should be /opacity
grep -rn 'bg-gradient-to-' src/      # Should be bg-linear-to-
grep -rn '!bg-' src/                 # Should be bg-!
grep -rn 'tailwind.config.js' .      # Should be @theme in CSS
grep -rn 'shadow-sm' src/            # Check intent: was v3 smallest, now v4 medium
```

## PRE-DELIVERY CHECKLIST

Before outputting ANY design:

### Visual ✓
- [ ] Clear visual hierarchy (1.5x size distinction, one primary action)
- [ ] Consistent spacing (4/8dp scale — NO arbitrary values)
- [ ] Proper color contrast (4.5:1 both light/dark mode)
- [ ] Typography scale consistent (12/14/16/18/24/32px)
- [ ] No emojis as icons (use SVG: Phosphor/Heroicons/Lucide)
- [ ] Semantic theme tokens used (NO ad-hoc hex values)
- [ ] Breathing room around content

### Technical ✓
- [ ] TypeScript types defined
- [ ] Responsive breakpoints tested (375/768/1024/1440)
- [ ] Accessible markup (ARIA labels, roles, states)
- [ ] Loading state handled (skeleton matching content shape)
- [ ] Error state handled (clear message + recovery path)
- [ ] Interactive states (hover, focus, active, disabled)
- [ ] Touch targets >= 44x44px (mobile)

### Integration ✓
- [ ] Ant Design components used correctly (if project uses Ant Design)
- [ ] Tailwind v4 syntax correct (@import, @theme, /opacity, bg-linear-to-)
- [ ] Imports organized
- [ ] Props interface defined
- [ ] Export default

### Accessibility ✓
- [ ] All meaningful images/icons have aria-labels
- [ ] Form fields have visible labels + hints + clear error messages
- [ ] Color is NOT the only indicator (add icon/text)
- [ ] `prefers-reduced-motion` respected
- [ ] Focus indicators visible (2-4px ring)
- [ ] Screen reader focus order matches visual order
- [ ] Heading hierarchy sequential (no level skip)

### Dark Mode ✓
- [ ] Primary text contrast >=4.5:1 in both modes
- [ ] Secondary text contrast >=3:1 in both modes
- [ ] Dividers/borders distinguishable in both modes
- [ ] Modal scrim 40-60% black for foreground legibility
- [ ] Colors desaturated 10-20% for dark mode
- [ ] Off-white text (#E0E0E0) not pure white (#FFFFFF)

## QUICK REFERENCE — Common Violations

| Violation | Severity | Fix |
|-----------|----------|-----|
| `outline-none` without focus alternative | Critical | `focus:ring-2 focus:ring-blue-500` |
| `placeholder='Email'` as only label | Critical | Add `<label htmlFor='email'>` |
| `w-6 h-6` clickable element | Critical | `min-w-[44px] min-h-[44px]` |
| `z-[9999]` | High | Use scale: `z-10`/`z-20`/`z-50` |
| `h-screen` on mobile | High | `min-h-dvh` |
| `@tailwind base` | Critical (v4) | `@import "tailwindcss"` |
| `bg-gradient-to-r` | High (v4) | `bg-linear-to-r` |
| `!bg-red-500` | High (v4) | `bg-red-500!` |
| `bg-[#hex]` arbitrary | Medium | Define in `@theme`, use token |
| Animating `width`/`height`/`top` | Medium | Use `transform`/`opacity` |
| Blank screen while loading | High | Skeleton matching content shape |
| Red border only for error | Critical | Add error text + icon |
| Emoji as structural icon | Medium | Use SVG icon (Heroicons/Lucide) |
| Pure white text on dark bg | Medium | `text-gray-100` not `text-white` |
| `type="text"` for email input | Medium | `type="email"` |
